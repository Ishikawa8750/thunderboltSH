# OpenBolt 阶段测试说明（先定义，暂不执行）

本文档用于明确每个阶段需要验证的内容。当前按要求仅创建文档，不执行测试。

## 阶段 A：链路与发现
- 目标：两端在雷电链路可互通并能发现对端。
- 需要验证：
  - 本机链路 IP 已配置。
  - mDNS 可发现对端设备并展示 IP。
  - 手动配置 IP 时界面能正确显示。

## 阶段 B：文件 API 基础能力
- 目标：文件服务具备列目录、查询状态、下载、上传能力。
- 需要验证：
  - `GET /api/fs/list` 返回目录项。
  - `GET /api/fs/stat` 返回 exists/mtime/size。
  - `GET /api/fs/download` 可流式下载。
  - `POST /api/fs/upload` 可写入远端目录。
  - 配置 `OPENBOLT_API_TOKEN` 后，未带 token 请求返回 401。
  - 带 `x-openbolt-token` 的请求可正常通过。

## 阶段 C：单向实时同步
- 目标：本地变更可增量同步到远端。
- 需要验证：
  - 新建文件后远端出现。
  - 修改文件后远端内容更新。
  - 防抖聚合生效（短时间重复写入只触发有限上传）。

## 阶段 D：删除与重命名同步
- 目标：不仅上传，还能把删除/重命名传播到远端。
- 需要验证：
  - 本地删除文件后远端同步删除。
  - 本地重命名后远端旧名删除、新名出现。
  - 不存在目标文件时删除接口返回幂等成功。
  - 本地重命名目录时，远端目录路径同步更新。

## 阶段 D.1：冲突策略（已定义）
- 目标：当远端更新更“新”时避免误覆盖。
- 当前策略：若远端 `mtime > 本地 mtime`，本地上传跳过并记录冲突日志（保留远端版本）。
- 需要验证：
  - 人为构造远端更晚修改时间后，本地修改不会覆盖远端。
  - 日志中出现 `conflict keep remote` 关键字。

## 阶段 E：跨平台联调（Windows <-> macOS）
- 目标：在真实链路中完成端到端同步。
- 需要验证：
  - Windows 到 macOS 同步成功。
  - macOS 到 Windows 同步成功（后续双向阶段）。
  - 网络抖动后恢复同步。

## 阶段 F：稳定性与恢复
- 目标：异常场景具备可恢复能力。
- 需要验证：
  - 对端离线后重连可继续同步。
  - API 端口被占用时有清晰提示。
  - 日志中能定位失败原因。
  - 失败任务会落盘到 `.openbolt/retry_queue.json`。
  - 应用重启后自动加载失败队列并继续重试。
  - 前端可查看重试队列 pending 数并支持清空。

## 阶段 F.1：双向防回环抑制
- 目标：减少双向同步时的重复回放。
- 当前策略：短时间窗口内相同路径+动作去重（本地事件抑制）。
- 需要验证：
  - 快速重复写入/回放不会触发无限重复上传。
  - 去重窗口外的真实变更仍会被同步。
  - `**/*.tmp`、`.openbolt/**` 等 glob 忽略规则按预期生效。

## 阶段 F.2：重试队列运维能力
- 目标：失败任务可观察、可干预。
- 需要验证：
  - 队列明细可显示每条任务 kind、attempts、target。
  - 支持单条任务立即重试。
  - 支持单条任务移除与全量清空。

## 阶段 G.1：真实双向同步管线
- 目标：双向模式下可实现对端变更拉取到本地。
- 实现策略：
  - 出站：本地 notify 事件 → 推送到对端（原有 outbound 管线）。
  - 入站：每 30s 轮询对端目录，下载比本地更新的文件（不含删除传播）。
  - 防回环：写入本地后将路径加入 `RemoteOriginCache`（10s 抑制），出站 watcher 命中则跳过。
- 需要验证：
  - 对端新建文件后，30s 内本地出现该文件。
  - 本地已较新的文件不被远端版本覆盖（mtime 比较）。
  - 入站写入不触发出站重新上传（原点缓存生效）。
  - 忽略规则在入站方向同样生效。

## 阶段 G.2：忽略规则预览
- 目标：用户在 UI 输入 ignore patterns 后可预览哪些本地路径会被命中。
- 需要验证：
  - 点击 "Preview Matches" 后返回命中的相对路径列表。
  - 修改 patterns 后预览结果自动清除（重新预览才更新）。
  - 目录不存在时命令返回错误提示。
  - 最多返回 200 条（防止过大扫描）。

## 阶段 G.3：重试队列筛选
- 目标：支持对队列明细按条件过滤展示。
- 需要验证：
  - 前端 Kind 下拉选 upsert/delete/move 可过滤对应条目。
  - Min attempts 数值输入可按最小尝试次数过滤。
  - 后端 `get_retry_queue_items` 接受 `kind_filter` 与 `min_attempts` 参数并正确筛选。

## 阶段 G：发布前检查
- 目标：可打包、可安装、权限说明完整。
- 需要验证：
  - Windows UAC 与 macOS 权限说明完整。
  - 安装后首次启动流程可用。
  - 基础功能回归通过。
